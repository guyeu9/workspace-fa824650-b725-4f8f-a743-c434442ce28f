## 常见错误预防

### 错误1：字段名错误
❌ 错误：使用 `chapter`, `scene_detail`, `choice`, `target`
✅ 正确：使用 `branch_title`, `content`, `option_text`, `target_branch_id`

### 错误2：状态变更格式错误
❌ 错误：
```json
"status_changes": {
  "金币": -10
}
```
✅ 正确：
```json
"status_changes": [
  {
    "attribute": "金币",
    "operation": "-",
    "value": 10
  }
]
```

### 错误3：条件格式错误
❌ 错误：
```json
"conditions": "hasItem(key)"
```
✅ 正确：
```json
"conditions": [
  {
    "type": "hasItem",
    "key": "item_id",
    "operator": "eq",
    "value": true
  }
]
```

### 错误4：branch_id 重复
❌ 错误：多个分支使用相同的 branch_id
✅ 正确：每个 branch_id 在 branches 数组中唯一

### 错误5：target_branch_id 不存在
❌ 错误：target_branch_id 指向不存在的分支
✅ 正确：确保所有 target_branch_id 对应的 branch_id 存在

### 错误6：空字符串显示
❌ 错误：effect 或 status_update 为空字符串 ""
✅ 正确：如果不显示，则不包含该字段或设置为 null

### 1. 结构化需求
将需求分为清晰的几个部分：
- 游戏基本信息
- 状态系统

### 2. 提供具体示例
为每个复杂概念提供示例：
- 状态变更示例

明确指定：
- 必填字段

- 检查 branch_id 唯一性

## 总结

使用这些提示词模板时，请记住：

1. **明确需求**：清楚地描述你想要的游戏功能
2. **提供示例**：如果可能，提供具体的例子
3. **指定格式**：明确要求使用标准字段名
4. **包含验证**：要求 AI 检查生成的 JSON
5. **迭代优化**：根据生成结果调整提示词

通过使用这些结构化的提示词，你可以更有效地与 AI 协作，创建高质量的游戏 JSON 文件。
