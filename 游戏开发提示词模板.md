  "branches": [
    {
      "branch_id": "[分支ID1]",
      "branch_title": "[分支标题1]",
      "content": "[分支内容1]",
      "options": [
        {
          "option_id": "[选项ID1]",
          "option_text": "[选项文本1]",
          "target_branch_id": "[目标分支ID]",
          "effect": "[效果文本]",
          "status_update": "[状态更新文本]",
          "status_changes": [
            {
              "attribute": "[属性名]",
              "operation": "[运算符]",
              "value": [变更值],
              "min": [最小值],
              "max": [最大值]
            }
          ],
          "conditions": [
            {
              "type": "[条件类型]",
              "key": "[物品ID]",
              "operator": "eq",

            }
          ],
          "consequences": [
            {
              "type": "[后果类型]",

## 常见错误预防

### 字段名称错误
- ❌ 错误：使用 `chapter`、`scene_detail`、`choices`、`id`、`choice`、`next_branch`
- ✅ 正确：使用 `branch_title`、`content`、`options`、`option_id`、`option_text`、`target_branch_id`

### 状态更新格式错误
- ❌ 错误：使用字符串格式 `"暴露度10%"`
- ✅ 正确：使用 status_changes 数组
```json
"status_changes": [
  {
    "attribute": "暴露度",
    "operation": "+",
    "value": 10,
    "max": 100
  }
]
```

### 缺少必填字段
- ❌ 错误：缺少 `branch_id`、`branch_title`、`content`、`option_id`、`option_text`、`target_branch_id`
- ✅ 正确：所有必填字段都已填写

### 分支ID重复
- ❌ 错误：多个分支使用相同的 `branch_id`
- ✅ 正确：每个 `branch_id` 在 branches 数组中唯一

### 目标分支不存在
- ❌ 错误：`target_branch_id` 指向不存在的 `branch_id`
- ✅ 正确：所有 `target_branch_id` 对应的 `branch_id` 都存在

## 开发建议

1. **从小开始**
   - 先创建简单的2-3个分支
   - 逐步添加更多分支和选项
   - 测试每个分支的跳转是否正确

2. **使用命名规范**
   - branch_id 使用英文小写和下划线
   - option_id 使用 opt_ 前缀
   - 物品ID 使用下划线分隔

3. **状态管理**
   - 为百分比属性使用"度"或"率"后缀
   - 使用 min 和 max 防止数值越界
   - 在 game_states 中定义所有初始状态

4. **测试验证**
   - 使用JSON验证工具检查格式
   - 测试所有分支跳转是否正常
   - 验证状态变更是否正确计算

5. **用户体验**
   - 使用 effect 提供即时反馈
   - 使用 status_update 说明状态变化
   - 使用 consequences 显示重要事件
   - 保持选项文本清晰简洁
