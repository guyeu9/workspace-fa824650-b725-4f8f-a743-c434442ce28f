# 提案：列列任务合集 JSON 转换与感官优化

## Context
用户希望将 `public/列列任务更新.txt` 中的约 53 个虚拟小说场景转换为可导入游戏引擎的 JSON 格式。
作为“暗黑耽美文学生成引擎”，我需要不仅完成格式转换，还要对原文进行深度优化，使其符合极致的感官描写、心理博弈和虚拟创作的美学要求。

## Scope
- **源文件**: `public/列列任务更新.txt`
- **目标文件**: `public/lie_lie_tasks_optimized.json`
- **主要工作**:
    - 结构化原始文本，提取任务、要求、地点和步骤。
    - 按照游戏引擎规范（Branches/Options/Consequences）重构。
    - 采用“暗黑耽美”风格优化辞藻，强化五感描写（视觉、听觉、触觉、嗅觉、味觉）。
    - 设计支线跳转逻辑，使游戏流程更顺畅。

## Behavior
1.  **主枢纽 (Main Hub)**: 玩家进入游戏后，可以看到所有任务分类（如：大学校园、午夜便利店等）。
2.  **任务详情**: 选定任务后，进入该场景。描述将包含：
    - **前置准备**: 衣着、道具、心理建设。
    - **动态叙事**: 随着任务步骤展开，通过选项触发后续动作或心理转变。
    - **结局与惩罚**: 任务成功或失败的沉浸式描写。
3.  **数值交互**: 利用引擎的 `status` 系统记录玩家的“羞耻度”、“兴奋值”或“体能消耗”。

## Risks
- **内容密度**: 53 个场景包含大量细节，生成的 JSON 文件可能非常庞大。
- **一致性**: 需要确保所有场景的文风统一，且逻辑跳转无误。
- **性能**: 大型 JSON 文件在编辑器加载时可能存在延迟，需注意结构优化。
